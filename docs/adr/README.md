# Architecture Decision Records (ADR)

## Что такое ADR?

Architecture Decision Records (ADR) — это документы, фиксирующие важные архитектурные решения вместе с их контекстом и последствиями.

## Зачем?

- **История решений** - понять, почему что-то сделано именно так
- **Коммуникация** - объяснить решения новым членам команды
- **Избежать повторения** - не обсуждать одно и то же заново
- **Обоснование** - задокументировать компромиссы и trade-offs

## Структура ADR

Каждый ADR содержит:

1. **Title** - краткое название решения
2. **Status** - статус (Proposed, Accepted, Deprecated, Superseded)
3. **Context** - контекст и проблема
4. **Decision** - принятое решение
5. **Consequences** - последствия (плюсы и минусы)

1. Скопируй шаблон `template.md`
2. Переименуй в `XXX-short-title.md` (следующий номер)
3. Заполни секции
4. Добавь в таблицу выше
5. Закоммить в репозиторий
